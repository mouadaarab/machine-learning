Algorithme Decision Tree (Arbre de Décision)

Qu'est-ce qu'un Decision Tree ?

Un Decision Tree (Arbre de Décision) est un algorithme d'apprentissage supervisé qui peut être utilisé pour les tâches de classification et de régression.

Il modélise les décisions sous forme d'arbre où chaque nœud interne représente un test sur un attribut, chaque branche représente le résultat du test, et chaque feuille représente une étiquette de classe.

Avantages

  • Interprétabilité : Facile à comprendre et à interpréter
  • Pas de préparation : Peu de préparation des données nécessaire
  • Flexibilité : Peut gérer des données numériques et catégorielles
  • Rapide : Prédictions rapides
  • Sélection automatique : Sélection automatique des caractéristiques importantes

Inconvénients

  • Surapprentissage : Tendance au surapprentissage
  • Instabilité : Petits changements dans les données peuvent créer un arbre très différent
  • Biais : Bias vers les caractéristiques avec plus de niveaux
  • Difficulté : Difficile à optimiser globalement

Paramètres Importants

  • max_depth : Profondeur maximale de l'arbre
  • min_samples_split : Nombre minimum d'échantillons pour diviser un nœud
  • min_samples_leaf : Nombre minimum d'échantillons dans une feuille
  • criterion : Fonction pour mesurer la qualité d'une division

Comment ça fonctionne ?

  1. Sélection de l'attribut : Choisir le meilleur attribut pour diviser les données
  2. Division : Diviser l'ensemble de données selon l'attribut sélectionné
  3. Récursion : Répéter le processus pour chaque sous-ensemble
  4. Arrêt : S'arrêter selon des critères prédéfinis (profondeur, nombre d'échantillons, etc.)
  5. Prédiction : Suivre le chemin dans l'arbre pour faire une prédiction

Différence avec Random Forest

Decision Tree :
  • Un seul arbre
  • Plus simple et rapide
  • Plus interprétable
  • Tendance au surapprentissage
Random Forest :
  • Ensemble de plusieurs arbres
  • Plus complexe mais plus robuste
  • Moins interprétable
  • Réduit le surapprentissage
